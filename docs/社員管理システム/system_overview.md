# 社員管理システム システム概要

## 1. システムの目的
本システムは、企業の従業員情報、所属部署、役職、および時給などの基本情報を統合的に管理するためのWebアプリケーションです。

## 2. 技術スタック
*   **バックエンド:** Python, Flask
*   **データベース:** SQLite, SQLAlchemy (ORM)
*   **フロントエンド:** HTML, CSS, JavaScript, Bootstrap 5

## 3. 主な機能
*   **社員管理機能**
    *   社員情報の登録・編集・詳細表示・削除（論理削除）
    *   時給設定機能（履歴管理対応）
    *   各種条件（社員コード、氏名/フリガナ曖昧、部署、役職、状態）による一覧の検索・絞り込み
*   **部署管理機能**
    *   部署の登録・編集・削除（論理削除）
    *   部署コード・部署名による検索
*   **役職管理機能**
    *   役職の登録・編集・削除（論理削除）
    *   役職コード・役職名による検索
*   **CSV連携機能**
    *   社員データのCSVエクスポート
    *   社員データのCSVインポート（新規登録・一括更新、未登録部署・役職の自動生成）

## 4. データベース設計（主要テーブル）
データの誤消去防止や履歴保持の観点から、マスターデータの削除は物理削除ではなく、各テーブルに設けられた `is_active` フラグを用いた**論理削除**（無効化）を採用しています。

*   **employees (社員テーブル):**
    社員コード、氏名、フリガナ、入社日、退職日、状態（在職/休職/退職）、所属部署ID、役職IDを管理。
*   **departments (部署テーブル):**
    部署コード、部署名を管理。
*   **positions (役職テーブル):**
    役職コード、役職名を管理。
*   **hourly_rate_history (時給履歴テーブル):**
    社員ごとの時給の変遷を管理。社員テーブルとは1対多のリレーションを持ち、適用開始日(`start_date`)と終了日(`end_date`)で履歴を保持する。

## 5. 今後の拡張方針
今後「住所」「電話番号」「配偶者の有無」など、社員に紐づく管理項目が増加する可能性があります。その際のアプローチとして以下を想定しています。

1.  **基本テーブルへの追加:**
    検索や一覧表示で頻繁に利用される項目であれば、既存の `employees` テーブルへ直接カラムを追加する。
2.  **詳細情報テーブルの分離:**
    項目数が多岐にわたる場合や、メイン画面での読み込みパフォーマンス低下を防ぐ場合は、新たに `employee_details` (社員詳細テーブル) を作成し、`employees` テーブルと1対1で連携させる設計とする。

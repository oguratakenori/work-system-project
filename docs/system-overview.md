# システム概要: 業務管理システム (work-system-project)

本プロジェクトは、製造現場等における業務を統合的に管理するためのシステム群です。
複数の独立したモジュールで構成され、各業務領域（社員、製品、作業実績、原価）を管理します。

## モジュール構成

### 1. 社員管理 (employee-management)
- 社員情報の登録・編集・削除。
- 所属部署や役職の管理。
- 社員ごとの時給管理。

### 2. 製品管理 (product-management)
- 製造対象となる製品情報の管理。
- 製品コード、名称、仕様などの保持。
- 製品単価管理。

### 3. 作業実績管理 (work-record-management)
- 日々の作業実績（誰が、いつ、どの部署で、どの製品を、何時間作業し、何個作ったか）の入力と集計。
- CSVレポート出力機能。

### 4. 原価計算・集計システム (expense-management)
- 各プロジェクトや部署に関連する原価の計算。
- 社員時給と作業実績に基づく労務費の自動集計。
- 製品単価と数量に基づく材料費・外注費の集計。
- ※ディレクトリ名は `expense-management` ですが、システムの役割に合わせて将来的に `cost-calculation` 等への名称変更を検討しています。

## システム連携

各モジュール間はCSVファイルによる非同期連携を基本とする。

**社員管理**
- 社員マスタCSVを出力

**製品管理**
- 製品マスタCSVを出力

**作業実績管理**
- 作業実績CSVを出力

**原価計算・集計システム**
- 各CSVを取り込み、原価計算および集計を行う

## データベース方針
- **開発時・プロトタイプ:** 構築の容易さと可搬性を重視し、SQLiteを採用。
- **将来的な運用:** データ量増加や同時接続数への対応を見据え、PostgreSQLへの移行が容易なようSQLAlchemy ORMによる抽象化を維持する。

## システムアーキテクチャ
- **Backend:** Python / Flask (work-record-management 等)
- **Database:** SQLite (開発時) / SQLAlchemy ORM
- **Frontend:** HTML5, CSS3 (Bootstrap 5), Jinja2 テンプレート

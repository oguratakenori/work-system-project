# 製品管理システム 概要・設計書

## 1. システムの目的
企業内の部署ごとに扱う製品のマスター情報を管理するためのシステムです。
※本システムは在庫管理や入出庫管理を目的としたものではなく、純粋な製品マスタの登録・管理に特化しています。

## 2. 主な機能
*   **製品管理 (CRUD):** 
    *   製品の登録、編集、削除（論理削除）
    *   同一製品コードでも部署が異なれば別製品として登録可能
*   **製品検索機能:** 製品コード、製品名、部署による絞り込み検索
*   **CSV連携:** 部署別の製品データのCSV一括インポートおよびエクスポート
*   **部署マスタ連携:** 部署データは別システム（社員管理システム）のデータベースを直接参照します。

## 3. データベース設計（主要テーブル）
*   **products (製品テーブル - 本システム内):**
    *   `department_id`: 部署ID（外部システム連携用キー）
    *   `product_code`: 製品コード
    *   `name`: 製品名
    *   `price`: 製品単価
    *   `registered_date`: 登録日
    *   `abolished_date`: 廃止日
    *   `notes`: 備考
    *   `is_active`: 有効フラグ(論理削除用)
    *   ※ ユニーク制約: `(department_id, product_code)`

*   **departments (部署テーブル - 社員管理システム側 DB):**
    *   本システムは SQLAlchemy の Binds 機能を使い、社員管理システムのDB（`employee_management.db`）を読み取り専用として参照します。

# 開発ロードマップ: 業務管理システム (work-system-project)

本プロジェクトの各機能の実装およびシステム連携、将来的な拡張に向けた段階的な開発計画です。

## 開発フェーズ

### フェーズ1: 基盤構築と単独モジュールの安定化
- **作業実績管理 (work-record-management) の完成** (現在進行中)
  - 基本的なCRUD機能、CSV出力、レポート表示の安定化。
  - 本モジュールの完成を最優先とし、他モジュールの開発基盤とする。

### フェーズ2: マスタ管理機能の拡充
1. **社員管理 (employee-management)**
   - 社員基本情報に加え、時給管理機能を実装。
2. **製品管理 (product-management)**
   - 製品基本情報に加え、製品単価管理機能を実装。

### フェーズ3: システム間連携と原価計算の導入
- **CSV連携の実装**
  - 各モジュールでのマスタCSVエクスポート機能。
- **原価計算・集計システム (expense-management) の構築**
  - 社員マスタ、製品マスタ、作業実績CSVのインポート機能の実装。
  - 集計ロジック（労務費・材料費計算）の実装とレポート表示。

### フェーズ4: 運用最適化とインフラ拡張
- **UI/UXのブラッシュアップ**
  - 全モジュールでの操作感の統一。
- **PostgreSQLへの移行検討**
  - データ量、同時アクセス数の増加に応じたDB移行の実施。

## システム作成順序
1. **作業実績管理** (実績収集の核となるため最優先)
2. **社員管理** (時給データ提供のため)
3. **製品管理** (単価データ提供のため)
4. **原価計算・集計システム** (各データの統合先として最後に構築)

## システム間連携の実施タイミング
- フェーズ3の開始時に、**作業実績管理からのデータエクスポート**を最初の連携ポイントとして実施。
- その後、社員・製品マスタのインポート機能を原価計算システム側に実装し、段階的に連携範囲を広げる。

## 将来のPostgreSQL移行検討タイミング
- **データ件数:** 作業実績レコードが数万件を超える見込みが立った段階。
- **ユーザー数:** 複数部署で同時に実績入力を行い、SQLiteの書き込みロックによるパフォーマンス低下が懸念される段階。
- **機能追加:** 複雑な分析クエリや集計処理において、PostgreSQLの高度な機能（ウィンドウ関数等）が必要となった段階。
